deployment_maintenance by davidrenne

A unique SVN, mysql and php deployment utility, for those businesses not converted to git yet. See below for features.

This deployment utility allows you to:

 

  1. Abstract multiple database schemas into “deployable regions” in order to ease sending up code and database script patches
  1. Also allows you to select multiple custom databases
  1. Hooks into multiple SVN repositories and branches to allow you to create code release packages
  1. Deletes, adds folders and modifies files into your region paths
  2. Parses checkin messages to relate to project Numbers
  3. Organize file ordering for release
  4. Control which branches of code can ONLY or can NEVER go into a region
  5. Warns you of certain matches in your code you are about to release
  6. Checks for PHP parse errors
  7. Creates a relationship between projects and blocks of code impacting the source and allows you to tag and organize blocks of code for testing purposes
  8. Links your projects/tickets to any ticketing system URL
  9. Allows you to rollback a package from any region
  10. Peer review OLD/NEW and visual HTML of differences
  11. Notifies production support and developers when all the code has been pushed
  12. Emails off hourly report from multiple branches of changes
  13. Keeps track of developers’ time by project basis (if project ID checked into each checkin message)
  1. Authenticates multiple developers
  2. Supports transactions, backing up tables and notifications in a custom SQL format
  1. Reports on execution time and errors for each revision
  1. Allows you to report and build custom output reports on multiple databases with the same schema in your regions listing
  1. Keeps track of all your SQL history to restore and reuse
  2. Combines the results from multiple databases into a single CSV file (even outputs affected counts on updates into a file across multiple databases)
  3. Allows you to add various data monitors that run once an hour
  4. Send Reports VIA URL to multiple parties
  5. Optionally polls MYSQL threads if you are thrashing your server with too many queries it will throttle
  6. Click a link to save and backup all the data into a permanent static HTML page
  7. Reads from SLAVE server and WRITES to master server
